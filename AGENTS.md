# AGENTS.md

このリポジトリで Application を実装するエージェント向けの基本ルールです。

## 目的
- 仕様に沿って、再現性のある実装を行う。
- 変更理由と検証結果を明確に残す。
- 破壊的変更を避け、小さく安全に進める。

## 実装ルール
1. **先に設計、次に実装**
   - 変更前に「目的 / 影響範囲 / 検証方法」を短く整理する。
2. **小さく分割して変更**
   - 1コミット1目的を原則とし、レビュー可能な粒度を保つ。
3. **既存資産を尊重**
   - 命名規則・ディレクトリ構成・記法を既存コードに合わせる。
4. **可読性優先**
   - コメントは「なぜ」を説明し、「なに」はコードで表現する。
5. **安全性の確保**
   - 機密情報をコミットしない（`.env`, 鍵, トークン等）。

## 検証ルール
- 変更に応じて最低1つ以上の実行可能なチェックを行う（例: lint/test/build）。
- コマンドと結果を記録し、失敗時は原因と回避策を明記する。

## ドキュメントルール
- 新規機能・重要変更は README または `docs/` に利用手順を追記する。
- 初回セットアップは、初学者が再現できるようにコマンドベースで記述する。

## 禁止事項
- 目的が不明確な大規模リファクタリング。
- テスト未実施のまま動作保証を断言すること。
- 根拠のない性能・セキュリティ主張。
