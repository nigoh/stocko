# Implementation Workflow Reference

## 実装前チェック
- 背景と目的が1段落で説明できる。
- In Scope / Out of Scope が分かれている。
- 受け入れ条件がチェックボックス形式で定義されている。

## 実装中チェック
- 1タスク1意図で変更されている。
- 命名と配置が既存構成に沿っている。
- 仕様変更を伴う場合、先にドキュメント差分がある。

## 実装後チェック
- 実行コマンドと結果を記録。
- 失敗テストがある場合、原因と暫定対応を明示。
- PR本文に背景・変更内容・検証・影響範囲を記載。

## Definition of Done
- [ ] 受け入れ条件を満たしている。
- [ ] 検証コマンドを1つ以上実行している。
- [ ] 必要なドキュメント更新が完了している。
